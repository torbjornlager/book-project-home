Proof-of-concept implementations

Two proof-of-concept implementations of a Prolog node are available, both written in SWI-Prolog. They exist to demonstrate that the ideas described in the book are realizable, not to serve as production-ready systems. Neither is complete, and both should be used in conjunction with the book.

Trinity demonstrator

The implementation of the Trinity demonstrator covers a large portion of the language and protocols, and gives a realistic picture of what functioning Web Prolog nodes look like in practice. The code is written in SWI-Prolog and JavaScript, often by help from various LLMs such as Anthropic's Claude, OpenAI's ChatGPT and Google's Gemini. There is a lot of code, and it's quite involved.

A simple node

This implementation is *much* smaller and is written for clarity. The code is meant to be read alongside the book. As a result, a number of features are absent or only partially sketched out. It is one of many starting points if your goal is to understand how Web Prolog nodes written by work from the inside.
